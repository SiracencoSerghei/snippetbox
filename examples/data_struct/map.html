Go Map Cheatsheet

What is a map?

map = key → value storage

scores := map[string]int{
    "alice": 100,
    "bob": 95,
}

Create

users := make(map[string]int)
cache := map[string]string{}

Set / Update

users["bob"] = 2
users["bob"]++

Read

value := users["bob"]

If key does not exist → returns zero value.

Check if key exists

value, ok := users["bob"]
if ok {
    fmt.Println(value)
}

Delete

delete(users, "bob")

Map of Structs + Embedded

type Address struct {
    City   string
    Street string
}

type User struct {
    ID   int
    Name string
    Address
}

users := map[string]User{
    "bob": {
        ID:   1,
        Name: "Bob",
        Address: Address{
            City:   "Brussels",
            Street: "Main St",
        },
    },
}

fmt.Println(users["bob"].City)

Embedding promotes fields, so City is accessible directly.

Important Behavior

map behaves like a hidden pointer.

m1 := map[string]int{"x": 1}
m2 := m1
m2["x"] = 50

fmt.Println(m1["x"]) // 50

Common Backend Use Cases

  • IP request counters
  • rate limiter
  • cache
  • headers
  • frequency counters
  • set / deduplication

Golden Rule

Known schema → struct

Dynamic keys → map

Set (via map)

Go has no built-in set type. Use map[T]struct{}.

visited := map[string]struct{}{}

visited["bob"] = struct{}{}
visited["alice"] = struct{}{}

_, ok := visited["bob"]
if ok {
    fmt.Println("exists")
}

delete(visited, "alice")

struct{} uses zero bytes, so this is the most memory-efficient set pattern.
